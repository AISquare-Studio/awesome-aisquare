# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any AISquare repository, please report it responsibly. **Do not open a public GitHub issue.**

Email us at [bots@aisquare.com](mailto:bots@aisquare.com) with:

- A description of the vulnerability
- Steps to reproduce it
- Any relevant logs or screenshots
- Your suggested fix (if you have one)

We will acknowledge receipt within 48 hours and aim to provide a resolution timeline within 5 business days.

## Supported Versions

| Repo | Supported |
|------|-----------|
| awesome-aisquare | Latest on `main` |
| AISquare-Studio-QA | Latest on `main` |
| django-ais | Pre-release — all reported issues will be addressed |

## Scope

This policy covers all repositories under the [AISquare-Studio](https://github.com/AISquare-Studio) GitHub organization.

Issues that are out of scope:

- Vulnerabilities in third-party dependencies (report these upstream, but let us know so we can update)
- Social engineering attacks
- Denial of service attacks

## Recognition

We appreciate security researchers who help keep AISquare safe. With your permission, we will credit you in the relevant release notes.
